# ClassFlowAI 개발 계획

이 문서는 ClassFlowAI의 다음 작업 순서와 완료 상태를 관리한다. 위에서 아래 순서로 진행하며, 구현과 필요한 검증이 끝난 항목만 완료로 표시한다.

## 상태 표시

- `[x]` 완료: 구현과 기본 검증이 끝나 현재 기준 코드에 반영됨
- `[ ]` 미완료: 구현 또는 실제 환경 검증이 남아 있음

## 우선순위

1. [x] **새 수업 시작 / 이전 수업 열기**
   - 현재 수업의 기록과 원본 이미지를 보존한 채 독립된 새 수업을 시작한다.
   - 기존 저장 폴더와 새 수업 폴더에서 이전 수업을 다시 열 수 있다.
   - 마지막으로 선택한 수업을 저장 위치별로 복원한다.
   - OCR/CAP 처리 중에는 안전한 수업 전환을 막는다.

2. [x] **현재 수업 초기화 시 이미지 삭제 여부 선택**
   - 기록만 초기화하거나 기록과 원본 이미지를 함께 정리하는 선택지를 제공한다.
   - 삭제 범위와 복구 불가 여부를 사용자에게 명확히 안내한다.
   - 이미지 유지 시 기존 이미지가 레코드로 자동 복원되지 않도록 제외 상태를 기록한다.
   - 일부 이미지 삭제가 실패하면 해당 파일을 유지하고 실패 경로와 결과를 안내한다.

3. [x] **CAP 원본 이미지 다시 복사**
   - 현재 선택한 CAP 캡처의 원본 이미지를 클립보드에 다시 복사하는 동작을 제공한다.
   - 기존 CAP 해석 결과 수동 복사와 혼동되지 않도록 버튼과 상태 문구를 구분한다.
   - 다시 복사한 이미지가 클립보드 감시에 의해 새 캡처로 중복 등록되지 않도록 해시 기준선을 갱신한다.

4. [x] **OCR·CAP 모델 연결 테스트**
   - 실제 NVIDIA API 키와 현재 기본 모델로 요청·응답 형식을 확인한다.
   - 성공, 인증 실패, 시간 제한, 모델 오류와 재시도 동작을 각각 확인한다.
   - API 키와 실제 사용자 데이터는 저장소에 남기지 않는다.
   - 테스트 결과와 재실행 방법은 `MODEL_CONNECTION_TEST.md`에 기록한다.

5. [ ] **실제 수업 캡처 20~50장 종단 테스트**
   - 캡처 감지부터 저장, OCR/CAP 처리, 탐색과 수업 전환까지 장시간 안정성을 확인한다.
   - 캡처 순서, 중복 감지, 레코드와 이미지 경로의 정합성을 점검한다.

6. [ ] **GPT ZIP → `notion_paste_package.zip` → Notion 붙여넣기 검증**
   - GPT 전달 ZIP 생성과 내부 이미지·타임라인·프롬프트 구성을 확인한다.
   - 생성된 `notion_paste_package.zip`의 Markdown, HTML과 복사 도구를 확인한다.
   - Windows CF_HTML 방식으로 실제 Notion 붙여넣기 결과를 검증한다.

7. [ ] **EXE 또는 설치 프로그램 배포 검토**
   - 현재 배치 파일 배포와 EXE·설치 프로그램 방식의 장단점을 비교한다.
   - Python, Tkinter, 필수 패키지 포함 방식과 업데이트·제거 흐름을 검토한다.
   - 사용자 설정, API 키와 수업 데이터가 업그레이드 중 보존되는지 확인한다.

8. [ ] **API 오류별 사용자 안내 개선**
   - 인증, 모델 ID, 네트워크, 사용량 제한, 시간 제한과 서버 오류를 구분한다.
   - 사용자가 바로 취할 수 있는 조치와 재시도 가능 여부를 상태창과 대화상자에 안내한다.
   - 로그에는 진단 정보를 남기되 API 키나 민감 정보는 기록하지 않는다.

## 학습카드 확장 단계

- [x] **1단계: GPT 학습카드 생성 규격**
  - GPT ZIP에 `STUDY_CARD_SPEC.md`를 포함하고 `study_cards.json`, `study_cards.md` 생성을 요구한다.
- [x] **2단계: 생성 JSON 자동 검증기**
  - 스키마 오류, 품질 경고, 근거 이미지 누락과 질문 중복을 Python API와 CLI로 검사한다.
  - 검증기는 카드를 자동 수정·삭제하거나 정답을 새로 생성하지 않는다.
- [x] **3단계: 카드 가져오기·검토**
  - `study_cards.json`과 `notion_paste_package.zip`을 검증한 뒤 수업별 작업공간으로 가져온다.
  - 기존 사용자 수정과 검토 상태를 보존하면서 중복을 병합하고 충돌을 보고한다.
  - 별도 검토 창에서 카드 수정, 승인, 제외와 검토 대기 전환을 제공한다.
- [x] **4단계: 오늘의 복습과 일정 관리**
  - 승인 카드의 오늘 복습, 답 확인과 사용자 직접 평가 흐름을 제공한다.
  - 카드 내용과 복습 상태·이력을 분리하고 재가져오기에도 일정을 보존한다.
  - 모름·어려움·보통·쉬움 평가로 다음 복습일을 결정적으로 계산한다.
- [ ] **파인만 답변 평가**
  - 사용자 설명을 핵심 요소와 비교하는 AI 자동 평가와 피드백을 구현한다.
- [ ] **실제 결과 품질 검증**
  - 실제 수업 및 정보처리기사 캡처로 생성된 `study_cards.json`을 검증하고 오탐·누락 기준을 조정한다.
- [ ] **학습카드 확장 전체 완료**
  - 실제 캡처 결과 검증이 끝나기 전에는 전체 기능을 완료로 표시하지 않는다.

현재 학습카드 확장 전체 상태는 **미완료**다. 오늘의 복습과 일정 관리는 구현되었지만 AI 자동 채점·파인만 답변 평가와 실제 수업 자료 종단 검증은 포함하지 않는다.

## 기존 기능 및 UI 보존 원칙

- 각 작업은 기존 OCR, CAP, 설정 저장, 전역 단축키, 클립보드 감지·복사와 GPT ZIP 생성 동작을 보존해야 한다.
- 사용자 설정, API 키, 수업 레코드와 원본 이미지를 명시적인 동의 없이 삭제하거나 초기화하지 않는다.
- 기존 저장 폴더와 레코드 형식은 가능한 한 하위 호환하고, 형식 변경이 필요하면 안전한 마이그레이션과 실패 시 복구 경로를 마련한다.
- 기존 메인 창, 미니 상태창, 버튼 위치·명칭과 단축키는 해당 우선 작업에 꼭 필요한 범위에서만 변경한다.
- 새 UI는 현재 흐름과 용어를 따르고 OCR/CAP 처리 상태 및 클립보드 결과를 명확히 구분한다.
- 변경 후 전체 Python 컴파일과 `app.py` import 검사를 기본으로 수행하며, 영향받는 기능은 실제 Windows 환경에서 추가 검증한다.
- 완료 표시는 코드 반영만으로 결정하지 않고 데이터 보존, 오류 처리와 관련 문서 갱신까지 확인한 뒤 변경한다.
