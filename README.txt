ClassFlowAI 사용 안내
======================

ClassFlowAI는 화면을 캡처하여 OCR 또는 이미지 해석으로 처리하고,
여러 캡처를 ChatGPT 정리용 ZIP으로 만드는 프로그램입니다.


1. 처음 실행
------------

1) ZIP 압축을 풉니다.
2) INSTALL_FIRST.bat을 실행합니다.
3) 설치가 끝나면 START_HERE.bat을 실행합니다.
4) 실행 준비 상태가 CMD 진행 바로 표시됩니다.
5) 설정에서 NVIDIA API 키를 입력합니다.


2. 기본 단축키
-------------

- Ctrl + Shift + S: 화면 캡처
- 휠클릭: OCR / CAP 전환
- Ctrl + 휠클릭: 감지 일시정지 / 다시 시작
- Shift + 휠클릭: ClassFlowAI 창 표시 / 최소화


3. OCR 모드
-----------

OCR 모드는 화면의 글자를 빠르게 추출하고,
필요하면 보정한 뒤 내용을 추론할 수 있습니다.

화면 캡처
→ OCR 실행
→ 오른쪽 결과창 표시
→ OCR 텍스트 자동 복사

OCR이 부정확하면 [OCR 보정 후 복사]를 사용합니다.
원본 이미지와 OCR 결과를 비교해 수정한 뒤 자동 복사합니다.

보정이 완료되면 [OCR 내용 해석] 버튼이 활성화됩니다.
이 버튼은 원본 이미지와 보정 OCR을 함께 추론 모델에 전달하여
내용의 의미, 구조와 흐름을 정리하고 자동 복사합니다.


4. CAP 모드
-----------

CAP 모드는 클립보드의 원본 이미지를 유지하면서
화면의 의미, 구조와 흐름을 해석합니다.

CAP 캡처 후에는 해석 텍스트를 자동 복사하지 않습니다.
따라서 Windows 캡처 도구가 클립보드에 넣은 원본 이미지가 유지됩니다.

화면 캡처
→ 원본 이미지 클립보드 유지
→ CAP 해석 결과를 오른쪽 결과창에 표시

해석 텍스트가 필요할 때만 [CAP 해석 복사]를 누릅니다.
[CAP 원본 이미지 복사]를 누르면 이전에 캡처한 CAP 원본 이미지를 클립보드에 다시 넣을 수 있습니다.
다시 복사한 이미지는 새 캡처로 중복 등록되지 않습니다.


5. 설정
------

[기본 / 모델]
- 캡처 저장 위치
- 단축키
- NVIDIA API 키
- OCR 모델 ID와 모델 페이지 URL
- CAP 추론 모델 ID와 모델 페이지 URL

기본 / 모델 탭은 별도 스크롤 없이 한 화면에 표시됩니다.
실제 API 요청에는 모델 ID가 사용됩니다.
모델 페이지 URL은 해당 모델 설명을 열기 위한 주소입니다.

[CAP 프롬프트]
- 이미지 한 장을 해석할 때 사용하는 프롬프트

[GPT ZIP 프롬프트]
- 여러 캡처의 수업 흐름을 복원하고 Notion 정리본을 만들 때 사용하는 프롬프트


6. GPT ZIP 사용
--------------

1) 수업 중 필요한 화면을 계속 캡처합니다.
2) [GPT ZIP파일 생성]을 누릅니다.
3) 생성된 ZIP을 ChatGPT에 첨부합니다.
4) ZIP에 포함된 프롬프트를 기준으로 수업 정리를 요청합니다.

기본 프롬프트는 다음을 우선합니다.

- 이미지 원본 우선
- 사용자가 정한 순서의 학습 흐름 복원
- 중복 캡처 통합
- 확인되지 않은 내용 추측 금지
- Notion용 HTML과 Markdown 생성
- Windows CF_HTML 클립보드 복사 도구 생성


7. 수업 관리
-----------

[새 수업 시작]
- 현재 수업의 기록과 원본 이미지를 그대로 보존합니다.
- 설정의 캡처 저장 위치 아래 lessons 폴더에 새 수업 폴더를 만듭니다.
- 새 수업은 캡처, 기록, 출력과 로그를 이전 수업과 분리해 저장합니다.

[이전 수업 열기]
- lessons 폴더에서 열 수업 폴더를 선택합니다.
- 기존 버전에서 사용하던 저장 폴더도 captures 또는 state 폴더가 있으면 열 수 있습니다.
- 마지막으로 연 수업은 다음 실행 때 다시 열립니다.

OCR 또는 CAP 처리가 진행 중이면 처리가 끝난 뒤 수업을 전환해야 합니다.

[수업 초기화]
- 예를 선택하면 현재 수업의 기록과 원본 이미지를 함께 삭제합니다. 삭제한 이미지는 복구할 수 없습니다.
- 아니요를 선택하면 기록만 초기화하고 원본 이미지는 유지합니다.
- 취소를 선택하면 아무것도 변경하지 않습니다.
- 삭제하지 못한 이미지가 있으면 해당 파일을 유지하고 경로를 안내합니다.

OCR 또는 CAP 처리가 진행 중이면 처리가 끝난 뒤 수업을 초기화해야 합니다.


8. 미니 상태창
-------------

- 파랑: OCR 대기
- 갈색: CAP 대기
- 주황: 진행 중
- 초록: 완료
- 빨강: 실패
- 회색: 일시정지

왼쪽 버튼으로 끌어서 이동할 수 있습니다.
더블클릭하면 메인 창을 표시합니다.
오른쪽 버튼을 누르면 메인 창 열기, 감지 일시정지·재개와 프로그램 종료 메뉴가 표시됩니다.
프로그램 종료는 확인창에서 확인한 경우에만 실행되며 저장된 수업 기록과 이미지는 유지됩니다.
Shift + 휠클릭은 메인 창이 앞에 있으면 최소화하고, 아니면 앞으로 표시합니다.


9. 주요 버튼
-----------

[새 수업 시작]
현재 수업을 보존하고 독립된 새 수업 폴더로 전환합니다.

[이전 수업 열기]
저장된 수업 폴더를 선택해 캡처와 처리 결과를 다시 불러옵니다.

[수업 초기화]
현재 수업 기록을 초기화하며 원본 이미지의 삭제 또는 유지를 선택합니다.

[GPT ZIP파일 생성]
현재 캡처와 사용자 학습 흐름 순서의 기록을 ZIP으로 만듭니다.

[HTML 흐름]
현재 캡처 흐름을 확인합니다.

[OCR 보정 후 복사]
OCR 결과를 이미지와 비교해 수정하고 자동 복사합니다.

[OCR 내용 해석]
OCR 보정이 끝난 캡처에서 활성화됩니다.
보정 OCR과 원본 이미지를 함께 분석하고 결과를 자동 복사합니다.

[CAP 해석 복사]
CAP 해석 텍스트를 사용자가 필요할 때 클립보드에 복사합니다.

[CAP 원본 이미지 복사]
현재 선택한 CAP 캡처의 원본 이미지를 클립보드에 다시 복사합니다.

[캡처 폴더 열기]
원본 캡처 이미지가 저장된 폴더를 엽니다.

[설정]
API 키, 저장 위치, 단축키, 모델과 프롬프트를 관리합니다.

[학습카드]
생성된 카드 JSON 또는 ZIP을 가져와 검토·수정·승인·제외합니다.
승인된 카드는 [오늘의 복습]에서 답을 확인한 뒤 모름·어려움·보통·쉬움으로 직접 평가합니다.
[AI로 설명 점검 (실험)]은 작성한 답에서 잘 설명한 부분과 빠진 부분을 참고용으로 보여줍니다.
AI 추천 등급은 자동 적용되지 않으며 최종 복습 평가는 사용자가 직접 선택합니다.
공식 정답이 확인되지 않은 카드는 AI 평가 화면에 별도 경고가 표시됩니다.
